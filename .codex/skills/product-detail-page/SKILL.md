# 상품 상세페이지 제작

쇼핑몰 상세페이지 작업을 두 단계로 처리한다.

- Phase 1: 카피라이팅과 한글 이미지 프롬프트 생성
- Phase 2: 생성된 섹션 이미지를 세로로 합쳐 최종 상세페이지 PNG 생성

## 언제 이 스킬을 쓸지

다음 요청에서 이 스킬을 사용한다.

- 상품 상세페이지를 처음부터 기획하거나 작성할 때
- 쿠팡, 스마트스토어, 자사몰용 상세페이지 문구가 필요할 때
- 이미지 생성용 한글 프롬프트가 필요할 때
- 이미 만든 섹션 이미지 여러 장을 하나로 합쳐야 할 때

사용자가 준 입력 상태에 따라 바로 맞는 단계로 들어간다.

- 상품명과 제품 정보만 있으면: Phase 1
- 섹션 이미지들이 이미 있으면: Phase 2
- 둘 다 있으면: Phase 1 결과를 먼저 만들고 Phase 2까지 이어서 진행

## 빠른 판단

작업을 시작하기 전에 아래를 확인한다.

1. 상품명
2. 제품 이미지 또는 참고 이미지
3. 사용자가 원하는 작업 범위

작업 범위는 세 가지 중 하나로 정리한다.

- "카피와 프롬프트만"
- "이미지 합치기만"
- "둘 다"

정보가 부족해도 멈추지 말고, 이미지와 상품명으로 추정 가능한 내용은 먼저 정리한다. 정말 필요한 경우에만 짧게 추가 확인한다.

## Phase 1: 카피와 한글 프롬프트 생성

### 입력

필수 입력:

- 상품명
- 제품 이미지 1장 이상

있으면 좋은 추가 입력:

- 핵심 특징 3가지
- 타겟 고객
- 가격대
- 차별점
- 톤앤매너

### 출력 목표

상세페이지를 7개 섹션으로 구성한다.

1. 히어로
2. 공감
3. 솔루션
4. 특징/혜택
5. 사용법
6. 신뢰요소
7. CTA

### 작성 원칙

- 문장은 짧고 바로 이해되게 쓴다.
- 특징은 반드시 고객 혜택으로 바꿔 쓴다.
- 막연한 표현보다 숫자, 상황, 결과를 우선한다.
- 과장 광고나 확인 불가능한 효능 표현은 피한다.
- 이미지 프롬프트는 반드시 한글로 쓴다.
- 이미지 프롬프트에는 제품 위치, 배경, 조명, 분위기, 텍스트 여백을 넣는다.

### 카피라이팅 체크 방식

- AIDA 흐름을 기본으로 쓴다.
- Feature -> Advantage -> Benefit 순서로 풀어쓴다.
- 첫 화면은 제품의 핵심 한 줄을 먼저 보여준다.
- 섹션마다 역할이 겹치지 않게 만든다.

### 기본 출력 형식

먼저 전체 구조 요약표를 보여준 뒤, 섹션별 상세 내용을 출력한다.

```markdown
## 전체 구조 요약

| 섹션 | 목적 | 핵심 메시지 |
| --- | --- | --- |
| 1. 히어로 | 첫인상 | ... |
| 2. 공감 | 문제 인식 | ... |
| 3. 솔루션 | 해결 제시 | ... |
| 4. 특징/혜택 | 장점 설명 | ... |
| 5. 사용법 | 사용 안내 | ... |
| 6. 신뢰요소 | 신뢰 확보 | ... |
| 7. CTA | 구매 유도 | ... |

## 섹션 1: 히어로

### 카피라이팅
- 메인 헤드라인:
- 서브 카피:
- 본문:

### 이미지 프롬프트
- 프롬프트:
- 스타일:
- 배경:
- 주요 요소:
- 분위기:
- 텍스트 공간:
```

### 이미지 프롬프트 규칙

모든 프롬프트는 아래 조건을 만족해야 한다.

- 한글로 작성
- 한 섹션당 한 개씩 제공
- 제품이 화면에서 차지하는 위치와 비율 명시
- 카피를 넣을 여백 위치 명시
- 필요하면 배경 소품을 1~2개만 넣어 과하지 않게 구성
- 쇼핑몰 상세페이지용이라는 목적을 직접 적는다

### 마무리 출력

섹션별 내용 뒤에 반드시 아래를 추가한다.

1. 나노바나나PRO 작업 순서
2. 이미지 생성 순서 추천
3. 사용자에게 다음 단계 안내

다음 단계 안내 예시:
"생성된 섹션 이미지들을 업로드하면 하나의 상세페이지 PNG로 합쳐드릴 수 있습니다."

## Phase 2: 상세페이지 이미지 합치기

사용자가 생성한 섹션 이미지들을 업로드하면 하나의 세로형 상세페이지로 합친다.

### 합치기 전에 확인할 항목

사용자가 따로 지정하지 않으면 아래 기본값을 사용한다.

- 이미지 순서: 파일명 순서
- 간격: 0px
- 최종 너비: 860px
- 배경색: 흰색
- 출력 형식: PNG

### 실행 방법

이미지 합치기는 반드시 `scripts/merge_images.py`를 사용한다. 직접 인라인 코드를 다시 쓰지 않는다.

입력 이미지가 크기가 달라도 괜찮다. 스크립트가 너비를 맞추고 세로로 이어 붙인다.

### 출력 규칙

- 파일명 기본값: `<상품명>_상세페이지.png`
- 상품명을 모르면 `detail_page.png`
- 완료 후 저장 경로를 함께 알려준다.

## 스크립트 사용

이미지 합치기가 필요할 때는 이 명령 형태를 사용한다.

```powershell
python scripts/merge_images.py --images 1.png 2.png 3.png --output detail_page.png --width 860 --gap 0
```

경로에 공백이 있으면 따옴표로 감싼다.

## 실패 시 처리

- 이미지 파일이 열리지 않으면 손상 여부와 경로를 확인한다.
- Pillow가 없으면 설치가 필요하다고 안내한다.
- 사용자가 순서를 지정하지 않았고 파일명도 불명확하면 업로드 순서를 그대로 사용한다고 밝힌다.

## 최종 응답 원칙

- 사용자가 바로 복사해 쓸 수 있게 결과를 정리한다.
- 긴 배경 설명보다 결과물과 다음 행동을 먼저 보여준다.
- Phase 2가 끝났다면 최종 파일 경로를 마지막 줄에 분명히 적는다.
