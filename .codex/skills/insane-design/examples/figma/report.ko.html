Design System Report

Figma — 흑백 극단 + --f-* component-slot

https://www.figma.com 의 CSS에서 직접 뽑은 디자인 토큰·타이포그래피·컴포넌트 스펙. 모노크롬 duo-theme, figmaSans variable font, 1px inset border 시그니처. 로고 5색은 UI에 0회.

Quick Start 5분 안에 Figma처럼 만들기 — 3가지만 하면 80%
1 · figmaSans + figmaMono /* 1. 폰트 — Figma 전용 커스텀 */ :root { --f-font-sans: "figmaSans", "figmaSans Fallback", "SF Pro Display", system-ui, sans-serif; --f-font-mono: "figmaMono", "figmaMono Fallback", "SF Mono", menlo, monospace; } body { font-family: var(--f-font-sans); font-weight: 400; font-size: 1.125rem; /* 18px! */ }
2 · duo-theme 흑백 반전 /* 2. 섹션별 bg 반전 */ .section-dark { background: #000000; color: #FFFFFF; } .section-light { background: #FFFFFF; color: #000000; } /* 같은 페이지 안에서 dark/light 섹션 교차 — Figma의 핵심 패턴 */
3 · 1px inset border 버튼 /* 3. Outlined button 시그니처 */ .btn-outlined { background: transparent; box-shadow: inset 0 0 0 1px currentColor; border: 0; border-radius: 8px; padding: 12px 24px; font: 540 16px/1 var(--f-font-sans); } /* drop shadow 없음. 1px inset이 전부. */
절대 하지 말아야 할 것: Figma 로고의 5색 (#0ACF83 green, #A259FF purple, #F24E1E red, #FF7262 peach, #1ABCFE blue)은 로고 SVG 전용이다. UI CSS에는 단 0회. 이 5색을 primary palette로 쓰면 colorful playful SaaS가 되어 Figma 브랜드 정체성이 완전히 사라진다.

03 · Typography

base 18px, bold는 540 — variable font 미세 조정

figmaSans variable font. 연속 weight 축으로 320~550 사용. 표준 700 대신 540/550이 bold 역할. base font-size 18px (16px 아님).

05 · Spacing

스페이싱(간격 체계) — 4개 토큰만, lego block 특화

custom spacing token이 4개뿐. 핵심은 --f-lego-block-padding: 5rem (80px). 일반 spacing은 Tailwind 유틸 추정.

lego block 패딩이 핵심: --f-lego-block-padding: 5rem (80px) — 모든 재사용 section의 inner padding. 그 외는 Tailwind utility 클래스 사용 추정.

06 · Border Radius

라디우스(모서리 반경) — 2px와 8px 단 2단

매우 제한적. 2px(chip/form/badge) + 8px(card/button). 8px 초과는 없음. "editor chrome minimal" 철학.

2-step radius 철학: 8px 이상을 거의 쓰지 않음. Figma의 "도구는 invisible" 철학 — 인터페이스가 장식을 줄이고 콘텐츠에 집중하도록.
07 · Shadows

drop shadow 없음 — 1px inset border로만 경계 표현

Figma는 box-shadow 변수를 0개 사용. 경계와 elevation은 inset 0 0 0 1px으로만. outlined button 시그니처.

1px inset border (outlined button)
버튼/카드 경계 — Figma 시그니처
inset 0 0 0 1px currentColor
drop shadow: 없음
elevation 표현에 drop shadow 사용 안 함
box-shadow 변수 0개 — 의도적 설계
1px inset border 패턴: box-shadow: inset 0 0 0 1px var(--f-text-color) — outlined button/card의 경계. currentColor 사용으로 dark/light section 모두 자동 적응. drop shadow 없음.

10 · Do / Don't

Do / Don't

지켜야 할 것과 피해야 할 것.

Do

  • Monochrome 흑백 극단: #000000 / #FFFFFF primary. 섹션마다 bg 반전 (duo-theme).
  • figmaSans + figmaMono Figma 전용 커스텀 폰트. 폴백은 SF Pro Display, SF Mono.
  • Variable font weight 540/550을 bold 역할로 — 표준 700 아님.
  • Body base font-size 18px (1.125rem, 최다) — 16px 아님.
  • 1px inset border button: box-shadow: inset 0 0 0 1px currentColor — outlined 시그니처.
  • Pastel block palette를 lego block section bg로 (#C7F8FB, #FFC9C1, #CB9FD2 등).
  • Radius 2-step: 2px(form/chip) + 8px(card/button). 이상 없음.
  • --f-* prefix slot 네이밍 체계 준수.

Don't

  • 로고 5색을 UI에 사용 — #0ACF83, #A259FF, #F24E1E, #FF7262, #1ABCFE는 로고 SVG 전용.
  • Inter 단독 — Figma는 figmaSans 전용 커스텀 폰트.
  • color-brand flat 네이밍 — Figma는 --f-* component-slot 체계.
  • 8px 초과 radius — Figma는 2px / 8px 2종만.
  • Drop shadow 기반 elevation — inset 1px border로만 경계 표현.
  • 단일 theme — Figma는 한 페이지 안에서 dark/light 섹션 반전.
  • 표준 font-weight: 700 — Figma는 variable font 540/550.
  • 16px base body — Figma는 18px base 사용.
