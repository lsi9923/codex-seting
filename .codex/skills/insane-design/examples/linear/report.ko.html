Design System Report

Linear — Marketing vs App 이중 팔레트

https://linear.app 이 실제로 서빙하는 CSS에서 직접 뽑은 디자인 토큰·타이포그래피·컴포넌트 스펙. 앱 UI와 마케팅 사이트가 서로 다른 indigo 팔레트를 운용하며, 거의 완전 블랙 배경에 7-tier 다크 surface 시스템을 쓴다.

Quick Start 5분 안에 Linear처럼 만들기 — 3가지만 하면 80%
1 · 폰트 /* Inter Variable + Berkeley Mono */ body { font-family: "Inter Variable", -apple-system, BlinkMacSystemFont, sans-serif; font-weight: 400; } code, pre, kbd { font-family: "Berkeley Mono", ui-monospace, monospace; }
2 · 다크 배경 /* 거의 순검정 */ :root { --color-bg-marketing: #010102; --color-bg-primary: #08090a; --color-text-primary: #f7f8f8; --color-text-secondary: #8a8f98; } body { background: var(--color-bg-marketing); color: var(--color-text-primary); }
3 · 브랜드 indigo /* marketing vs app 분리 */ :root { --color-brand-marketing: #4354B8; /* 124회 실사용 */ --color-brand-app: #5E6AD2; /* 앱 UI 전용 */ }
절대 하지 말아야 할 것: #5E6AD2 단일 값을 모든 곳에 쓰지 말 것. Linear는 앱 UI와 마케팅 사이트가 서로 다른 보라 팔레트를 쓴다. 마케팅 사이트에서는 #4354B8(훨씬 진한 indigo)이 실제로 124회 쓰이며, 앱 토큰 --color-indigo: #5e6ad2는 5회만 등장한다.

03 · Typography

타이포그래피 — Product + Marketing 이중 스케일

Linear는 두 개의 병렬 타이포 스케일을 운용한다. Product UI용(--font-size-*)과 marketing 대형 히어로용(--title-N-*). Variable weight 450/550 같은 non-표준 값 사용.

Font stack
Display / Body Inter Variable (variable font, weight 100-900 전 범위)
Code / Mono Berkeley Mono (유료 — JetBrains Mono 아님)
System fallback -apple-system, BlinkMacSystemFont, Segoe UI, Roboto
Non-standard weights 450, 550 — Inter Variable만 가능한 mid-weight
Berkeley Mono는 유료 폰트. U.S. Graphics Company 제작. OSS 대체로는 JetBrains Mono, Commit Mono, Geist Mono가 있지만 x-height와 메트릭이 다르다.

07 · Shadows

섀도우는 5-layer 미세 stacking

Linear는 의도적으로 drop shadow를 최소화한다. --shadow-low/medium/high가 실제로는 none으로 alias 되어 있으며, elevation은 border + 반투명 배경 + 5-layer 미세 stacking으로 표현한다.

--shadow-stack-low
카드 elevation (5-layer stacking)
0px 8px 2px rgba(0,0,0,0) + 4개 layer
--shadow-low / medium / high
alias → none (의도적)
shadow: none
철학: 단일 box-shadow: 0 4px 16px rgba(0,0,0,0.4) 같은 거친 drop shadow는 Linear 철학에 반한다. 반드시 5-layer 미세 stacking을 쓰거나, 아예 shadow 없이 border + rgba(255,255,255,0.05) 반투명 배경으로 elevation을 표현할 것.

11 · Do / Don't

Do / Don't

지켜야 할 것과 피해야 할 것.

Do

  • 브랜드 보라를 marketing vs app으로 분리. marketing은 #4354B8, app은 #5E6AD2.
  • 다크 배경은 7-tier(marketing/primary/panel/secondary/tertiary/quaternary/quinary). 단일 bg 금지.
  • Berkeley Mono(유료)가 진짜 코드 폰트. OSS 대체 시 JetBrains/Commit/Geist Mono.
  • Pastel highlight quartet(#F79CE0/#55CDFF/#89D196/#FFC47C)을 feature badge에 할당.
  • 타이포는 product(--font-size-*) + marketing(--title-N-*) 병렬 스케일. marketing 히어로는 56-64px.
  • Letter-spacing optical compensation(-0.011em ~ -0.022em). 제목이 커질수록 타이트.
  • Shadow는 5-layer 미세 stacking만. 거친 drop shadow 금지.

Don't

  • #5E6AD2 단일 값을 marketing hero에 쓰지 말 것 → 실제는 #4354B8(124회 vs 5회).
  • JetBrains Mono를 Linear 코드 폰트로 쓰지 말 것 → 실제는 Berkeley Mono.
  • #171723 같은 푸른 기 다크 bg 쓰지 말 것 → Linear는 #08090a/#010102(거의 완전 블랙).
  • 단일 box-shadow: 0 4px 16px rgba(0,0,0,0.4) 금지 → 5-layer stacking 또는 shadow 없음.
  • #F2994A orange 쓰지 말 것 → 실제는 #E5591D(훨씬 진함, 76회).
  • 배경 단일 톤 금지 → 7-tier 배경 시스템 필수.
  • Title 스케일을 24/32/48로만 구성하지 말 것 → marketing은 17/20/24/32/40/48/56/64px 8단계.
