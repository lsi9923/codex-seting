Design System Report

Toss — TDS

https://toss.im 이 실제로 서빙하는 CSS 에서 직접 뽑은 디자인 토큰·타이포그래피·컴포넌트 스펙. 모든 값은 CSS 커스텀 프로퍼티와 클래스 정의를 파싱한 결과이고, AI 추론이나 추측은 섞이지 않았다.

Quick Start 5분 안에 Toss처럼 만들기 — 3가지만 하면 80%
1 · 폰트 + weight 500 body { font-family: "Toss Product Sans", "SF Pro KR", -apple-system, sans-serif; font-weight: 500; }
2 · 배경 + 텍스트 :root { --bg: #F2F4F6; --fg: #191F28; } body { background: var(--bg); color: var(--fg); }
3 · 브랜드 블루 :root { --brand: #3182F6; } .btn { background: var(--brand); color: #fff; font-weight: 600; border-radius: 12px; }
절대 하지 말아야 할 것 하나: 배경을 #FFFFFF 순백으로 두는 것. Toss의 기본 배경은 #F2F4F6 cool gray다. 순백 바탕에 놓으면 Toss 특유의 카드 위 카드 레이어링 구조가 무너진다.

테크 스택

03 · Typography

1px 단위 촘촘한 20단 타이포 스케일

Toss Product Sans를 기본으로, t1~t7 주 스케일과 st1~st13 보간 스케일로 구성. Weight 클래스 없이 컴포넌트별로 직접 font-weight 지정하는 구조.

05 · Spacing

4px 기반 암묵 그리드, 토큰 변수 없음

Toss는 명시적 spacing 토큰을 사용하지 않는다. 모든 간격은 컴포넌트별 직접 px 값 지정이며, 4px 기반 그리드를 암묵적으로 따른다.

네이밍 규칙: spacing 전용 CSS 변수 없음. 컴포넌트 정의에서 직접 padding, margin, gap 값을 px 단위로 작성.

07 · Shadows

3-layer rgba 섀도우

모든 elevation은 outline ring + spread + edge 3겹 구조. rgba 기반이라 다크 모드에서도 자연스럽게 작동한다.

elevation-low
기본 카드, 조용한 float
3-layer · low opacity
elevation-high
바텀시트 CTA, 모달
3-layer · high opacity
elevation-modal
풀 모달 다이얼로그
3-layer · #000 base
패턴: inset shadow는 inset 0 0 0 1px 형태로 포커스 링(rgba(0,0,33,0.16))과 가벼운 보더 대용으로 사용된다.
08 · Motion

100ms ease-in-out 기본 리듬

대부분의 인터랙션은 0.1s ease-in-out. 배경 전환은 0.2s ease. 프로그레스 바 등 강조 전환만 0.5s 사용.

button-feedback .1s ease-in-out 버튼 hover/active 피드백
state-change .1s ease-in-out color, bg, border, shadow, opacity 전체 상태 전환
background-soft .2s ease 배경색 부드러운 전환
transform .2s ease-in-out 슬라이드, 회전 (accordion arrow 등)
progress .5s ease-in-out 프로그레스 바 확장
input-morph .1s cubic-bezier(.4,0,.2,1) 인풋 라벨 사이즈/무게/색상 변형
generic .25s ease-in-out 일반 전환 기본값
09 · Components

4가지 type, 4가지 size의 버튼 시스템

TDS 컴포넌트의 핵심은 버튼. type(primary/danger/dark/light)과 style(default/weak) 조합, size(tiny/medium/large/big) 축으로 전체 버튼 매트릭스를 구성한다.

.button Primary CTA — 4 types × 2 styles × 4 sizes

color: #FFFFFF
font-weight: 600
border-radius: 12px (large)
transition: .1s ease-in-out Weak variant: bg #E8F3FF, text #3182F6. Danger: bg #F04452. Dark: bg #4E5968.

12 · Do / Don't

자주 틀리는 것들

Toss 디자인을 구현할 때 실제 CSS에서 검증된 규칙.

DO

  • 배경은 #F2F4F6 cool gray, 그 위에 #FFFFFF 카드를 올려라 — Toss의 레이어 구조 핵심
  • 브랜드 블루 #3182F6는 CTA 버튼과 primary 링크에만 써라
  • 버튼 사이즈에 맞는 radius를 쓸 것: tiny=6, medium=8, large=12, big=16
  • Grey ramp 5단계 텍스트 위계를 지킬 것: #191F28 → #333D4B → #4E5968 → #8B95A1 → #B0B8C1
  • 본문 font-weight는 500이 기본. 400은 서브텍스트, 600은 버튼, 700은 제목에만
  • shadow는 항상 3-layer (outline + spread + edge) 구조
  • badge 배경은 해당 색상의 rgba(*, 0.16) opacity 패턴

DON'T

  • 배경에 #FFFFFF 순백을 쓰지 마라 — Toss의 기본 page background는 #F2F4F6
  • 브랜드 블루를 배경색으로 넓게 깔지 마라 — 블루는 포인트 컬러로만 사용
  • font-weight: 400을 body 기본으로 두지 마라 — Toss는 500이 normal
  • Grey scale에서 #000000 순흑 사용 금지 — 가장 진한 텍스트도 #191F28
  • 단일 레이어 box-shadow로 elevation을 재현하지 마라 — 3-layer가 아니면 깊이감이 다름
  • Toss Product Sans 없이 대체 폰트를 쓸 때 letter-spacing을 건드리지 마라 — 기본값이 맞다
