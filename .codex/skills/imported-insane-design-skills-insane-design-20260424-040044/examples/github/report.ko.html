--base-size-N = N px 그대로

토큰 숫자가 px 값과 일치. 8pt 그리드 기반, 4pt 세부 단위 포함.

마케팅 vs 제품 UI — 1280px 컨테이너 공통

Hero(마케팅 단일컬럼)와 Repository page(파일트리 + 컨텐츠 + 메타 3단) 두 레이아웃 베이스

Hero (marketing)
중앙 정렬 단일 컬럼 또는 2-column 스플릿. #0d1117 + SVG 그라디언트 오비트. H1: 48-64px / weight 800 / tracking -0.02em.
/* max-width: 1280px */
h1 {
  font-size: 48px;
  font-weight: 800;
  letter-spacing: -0.02em;
  line-height: 1.15;
}
Repository page (product UI)
좌측 파일 트리 + 중앙 컨텐츠 + 우측 메타 패널. Container max-width 1280px. Gutter: --base-size-24.
/* pageLayout-contentWidth-xxlarge */
.layout {
  max-width: 1280px;
  display: grid;
  grid-template-columns:
    240px 1fr 296px;
  gap: var(--base-size-24);
}
Section Rhythm
섹션 수직 여백은 64px. Box 컴포넌트는 6px 라디우스 + 1px border로 통일.
section {
  padding:
    var(--base-size-64)
    var(--base-size-24);
}
.Box {
  border-radius: 6px;
  border: 1px solid
    var(--borderColor-default);

Button Primary — 녹색 CTA
Primary 버튼은 파란색이 아닌 녹색. accent/link 역할만 파란색. 5px 16px 패딩, 6px 라디우스, weight 500.

자주 틀리는 것들

라이트 v1 잔재, 파란 CTA, 단일 hex 패턴 — Primer 재현 시 반복되는 실수

DO
  • 다크/라이트 두 모드 모두 문서화. data-color-mode 속성 기반 스위치.
  • Semantic color를 3단계 (emphasis / muted / fg)로 구성. 단일 hex 금지.
  • Heading에 Mona Sans Header Fallback 별도 지정. Body는 Mona Sans Fallback.
  • Primary 버튼은 녹색 #238636. 파란색은 accent/link.
  • done purple (#8957e5)은 merged PR 전용.
  • floating-* shadow 첫 레이어: 0 0 0 1px #3d444d hairline ring.
  • Spacing: --base-size-{N} 포맷, 숫자 = px 값.
  • Variable weight 활용: 550 (label), 800 (heading), 200 (thin accent).
DON'T
  • #0969DA (라이트 v1 accent)를 다크 사이트에 쓰지 말 것.
  • #24292F를 body text color로 쓰지 말 것 — 다크는 #f0f6fc.
  • #2DA44E (라이트 success)를 다크에 쓰지 말 것 — 다크는 #238636.
  • Primary 버튼을 파란색으로 만들지 말 것.
  • Semantic color를 단일 hex로 표현하지 말 것 — emphasis/muted/fg 튜플 필수.
  • 단일 box-shadow 사용 금지 — floating-*는 최소 2-3 레이어 합성.
  • font-weight: 700만 쓰지 말 것 — 200/450/550/800이 실제 시그너처.
  • #0d1117 대신 #000 사용 금지 — Primer canvas는 살짝 파란 기가 있다.
