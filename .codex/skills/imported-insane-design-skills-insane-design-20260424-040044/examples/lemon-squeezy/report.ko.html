Design System Report

Lemon Squeezy — 진한 보라 + 레몬 노랑, Webflow

https://www.lemonsqueezy.com 이 실제로 서빙하는 CSS에서 직접 뽑은 디자인 토큰·타이포그래피·컴포넌트 스펙. Webflow 빌더 출력이라 공식 토큰 시스템 없이 대부분 하드코딩된 값이며, 디자이너가 직접 지정한 6개의 수동 CSS 변수가 전부다.

Quick Start 5분 안에 Lemon Squeezy처럼 만들기 — 3가지만 하면 80%
1 · 폰트 /* Webflow Cloud Font */ body { font-family: "Circularpro book", "Circular Pro", "Inter", -apple-system, sans-serif; font-weight: 400; }
2 · 라이트/다크 혼합 /* 라이트 surface가 dominant */ :root { --bg-light: #ffffff; --bg-soft: #f7f7f8; --bg-dark: #121217; --fg-dark: #121217; --fg-muted: #6c6c89; --border: #d1d1db; }
3 · 브랜드 색상 /* 진한 보라 + 레몬 노랑 */ :root { --brand-purple: #5423E7; --brand-purple-2: #7047eb; --brand-yellow: #ffc233; --brand-yellow-2: #ffd266; }
절대 하지 말아야 할 것: 브랜드 보라를 #7C3AED(Tailwind Violet-600)로 쓰지 말 것. 실제 Lemon Squeezy 보라는 훨씬 더 진한 #5423E7(12회 실사용)다. 둘 다 보라처럼 보이지만 L* 차이가 15 이상이라 나란히 놓으면 완전 다른 브랜드로 보인다.

03 · Typography

타이포그래피 — weight 500이 강조 기본값

폰트 스택, 굵기 정의, 타이포그래피 스케일. Weight 실제 분포: 500(35회) > 400(31회) > 700(11회). 500을 강조 기본값으로 쓰는 것이 Lemon Squeezy의 시그너처다.

Font stack
Display / Body Circular Pro Book (Lineto 유료, Webflow Cloud Font로 로드)
Secondary / Form Inter — 폼 필드와 label에 한정 사용
Code JetBrains Mono
Icon webflow-icons (Webflow 내장 아이콘 폰트)
Weight 분포 500(35회) · 400(31회) · 700(11회)
라이선스 주의: Circular Pro는 Lineto 유료 폰트. OSS 대체로는 TT Commons Pro, Visby CF 같은 geometric grotesque가 가장 가깝다. Inter로는 재현 불가 — Circular의 round terminal(터미널 획이 둥근 형태)이 시그너처.

07 · Shadows

섀도우 — 노란 inset이 시그너처

Webflow 빌더는 shadow를 변수화하지 않는다(0개). 모든 그림자는 인라인. 노란 inset underline이 Lemon Squeezy 특유의 디테일이다.

노란 inset underline은 Lemon Squeezy의 시그너처 디테일이다. 링크나 강조 텍스트 하단에 2px 노랑 줄이 inset box-shadow로 그려진다. 일반 text-decoration: underline 대신 반드시 inset 0 -2px 0 0 #FFC233 형태로 구현할 것.
08 · Layout

레이아웃 패턴

반복되는 레이아웃 구조. 히어로는 다크, 피처 섹션은 라이트.

Hero

  • Layout: 중앙 정렬, 다크 배경 (#121217) + 보라/핑크 그라디언트 orb
  • H1: 40px~56px / weight 700 / tracking -0.02em

11 · Do / Don't

Do / Don't

지켜야 할 것과 피해야 할 것.

Do

  • 브랜드 보라는 #5423E7(진한 보라). 보조로 #7047EB.
  • 폰트는 Circular Pro Book(Webflow Cloud Font). Inter는 폼 필드 보조 역할만.
  • 링크/강조에 노란 inset 밑줄(inset 0 -2px 0 0 #FFC233) 사용. 기본 text-decoration 대신.
  • 라이트/다크 혼합 디자인. 히어로는 다크(#121217), 피처 섹션은 라이트. 라이트 surface가 dominant.
  • Radius는 20px가 기본(feature card). 버튼은 12px, 히어로 blob은 96~200px.
  • Weight 500을 강조 기본값으로. 800/900은 거의 쓰지 않음.
  • 노란색 #FFC233을 액센트로, #FFD266(밝은)을 하이라이트 블록으로 구분 사용.

Don't

  • 브랜드 보라를 #7C3AED(Tailwind Violet)로 쓰지 말 것 → 실제는 #5423E7, 톤이 훨씬 진하다.
  • Inter 단일을 주 폰트로 쓰지 말 것 → Circular Pro가 주, Inter는 폼 보조.
  • Dark-only 디자인으로 만들지 말 것 → 라이트 surface가 실제로 더 많이 쓰인다.
  • #1E1E2E / #2D2D3F 허구 다크 값 사용 금지 → 실제 다크는 #121217.
  • #3898EC(Webflow 기본 focus)를 브랜드 색으로 쓰지 말 것 → Webflow editor 디폴트.
  • 단일 text-decoration: underline 금지 → 반드시 노란 inset shadow 패턴 사용.
  • Display weight를 800으로 고정 금지 → 실제 빈도는 700/500 주도.
  • BEM semantic 토큰 시스템을 기대하지 말 것 → Webflow 빌더 출력이라 토큰 0개, 대부분 hardcoded.
